# PR 描述输出示例

本文档展示正确和错误的 PR 描述格式，帮助理解用户的严格要求。

## ❌ 错误示例 1：包含多余章节

```markdown
## 概述
本次重构优化了 Casbin 权限系统...

## 主要变更

### 核心重构
- **Casbin Enforcer 初始化逻辑重构**
  - 移除独立数据库连接创建
  - 接入 Ent 共享连接池

## 技术收益
- **资源优化**: 消除重复数据库连接
- **性能提升**: 复用连接池减少连接建立时间

## 测试计划
- [x] 单元测试覆盖核心重构逻辑
- [ ] 集成测试验证权限检查功能
```

**问题**：包含了"概述"、"技术收益"、"测试计划"等章节，违反了"只输出主要变更"的要求。

---

## ❌ 错误示例 2：包含统计和标识

```markdown
## 主要变更

### 1. 通知管理模块
- 新增 ListNotification 接口

### 2. 市场观点接口优化
- 市场观点列表接口增加判空处理

### 文件变更统计
- 39 个文件变更
- 新增 5160 行代码

### 相关提交
- 7ff47b9 chore: 合并 test 分支

---

🤖 Generated with Claude Code
```

**问题**：在"主要变更"后添加了"文件变更统计"、"相关提交"、分隔线和生成标识。

---

## ✅ 正确示例 1：通知系统 PR

```markdown
## 主要变更

### 1. 消息通知系统
- **新增消息列表查询接口** `GET /api/v1/notification`
  - 支持按状态筛选（pending/completed）
  - 支持按类型筛选（opinion/feedback/order）
  - 支持内容关键词搜索
  - 支持时间排序
  - 分页查询功能
- **数据模型**
  - 新增 Notification 实体及完整 CRUD 操作
  - 包含消息内容、状态、类型、接收者、关联对象等字段
- **完整三层架构实现**
  - Service 层：接口参数转换与业务编排
  - Biz 层：业务逻辑处理
  - Data 层：数据持久化与查询

### 2. 市场观点模块
- 市场观点列表接口增加判空处理
- 市场观点细节功能开发完善
```

**正确之处**：
- 只有"## 主要变更"这一个章节
- 按模块分组，层次清晰
- 没有任何多余内容
- 描述结束后立即停止

## ✅ 正确示例 2：Bug 修复 PR

```markdown
## 主要变更

### 1. 用户认证模块
- **登录流程空指针修复**
  - 用户信息获取前增加空值检查
  - 优化用户查询逻辑，处理用户不存在场景
  - 数据库查询增加错误处理
- **测试覆盖**
  - 新增空用户场景测试
  - 新增无效 token 测试
  - 新增数据库异常处理测试
```

**正确之处**：
- 简洁明了，直击问题核心
- 包含修复内容和测试覆盖
- 没有"问题背景"、"变更统计"等额外章节

---

## ✅ 正确示例 3：重构 PR

```markdown
## 主要变更

### 1. Casbin 权限系统重构
- **连接池复用**
  - Enforcer 初始化改用 Ent 共享连接池
  - 新增自定义 Adapter 支持 Ent 连接池
  - 导出数据库连接池供 Casbin 使用
- **性能优化**
  - 权限检查中间件减少冗余查询
  - 新增连接池监控指标和健康检查
- **测试完善**
  - 新增 Enforcer 初始化单元测试
  - 覆盖连接池复用场景
```

**正确之处**：
- 重构的核心改进一目了然
- 分类清晰（连接池、性能、测试）
- 没有"技术收益"、"影响范围"等章节

---

## 关键原则

1. **唯一章节**：PR 描述只能有"## 主要变更"这一个二级标题
2. **模块分组**：使用"### 1. 模块名"、"### 2. 模块名"组织内容
3. **立即停止**：最后一个变更点结束后，立即停止输出
4. **零多余**：不要任何概述、总结、统计、标识、分隔线

记住：用户明确要求"只输出主要变更"，其他一切都是噪音。
